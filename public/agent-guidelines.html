üîß 2. Bug Fixing - De Juiste Manier

‚ùå FOUT: Pleister plakken

// Bug: prijs toont NaN
// FOUT - dit is een pleister:
const price = isNaN(product.price) 
  ? 0 
  : product.price

Dit verbergt het probleem maar lost het niet op!

‚úÖ GOED: Oorzaak fixen

// Bug: prijs toont NaN
// GOED - zoek WAAROM:
// 1. Check waar price vandaan komt
// 2. Fix de bron (database/API)
// 3. Validatie bij de bron

Dit voorkomt dat de bug terugkomt!

Stappen voor Bug Fixing

Stap Actie Vraag jezelf
1. Reproduceer Kun je de bug zelf zien? Heb ik een screenshot/video?
2. Vind oorzaak Waar komt het probleem vandaan? Welke code veroorzaakt dit?
3. Begrijp context Waarom is de code zo geschreven? Wat kan er kapot gaan?
4. Fix root cause Los het ECHTE probleem op Is dit een echte fix of pleister?
5. Test Werkt de fix? Is er niets anders kapot?
6. Commit Duidelijke message Begrijpt iemand anders dit?

Commit Message Format

fix: [kort probleem] - [korte oplossing]

Probleem:
- Beschrijf wat er mis was

Oorzaak:
- Beschrijf waarom het mis ging

Oplossing:
- Beschrijf wat je hebt veranderd

Test:
- Beschrijf hoe je hebt getest

‚úÖ GOED: Kleine commits

  • 1 commit = 1 logische verandering
  • Makkelijk te reviewen
  • Makkelijk terug te draaien

‚ùå FOUT: Mega commits

  • "Fixed everything" met 50 files
  • Niemand kan dit reviewen
  • Onmogelijk terug te draaien

Voor Je Commit

# 1. Check wat je hebt veranderd
git status
git diff

# 2. Build/lint check
npm run build
npm run lint

# 3. Alleen relevante files toevoegen
git add [specifieke files]

# 4. NIET doen:
git add .  # Voegt ALLES toe, ook troep!
